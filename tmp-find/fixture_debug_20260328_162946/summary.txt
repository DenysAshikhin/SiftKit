## Unlocks.gd Analysis

### 1. Progression/Unlock Data Owned

Unlocks.gd is a Godot GDScript file that extends Node and contains:

- **OBJECT_KEYS**: A const dictionary tracking 50+ game progression metrics including:
  - Resource counters: drolium, slag, ochium, yetrium, thorium, osnium (combat and total variants)
  - Research: research_points, research_points_total, permanent_upgrades_count
  - Ship counters: drone_num_created, support_num_created, frigate_num_created
  - Damage/kill tracking: mothership_dealt_damage, drones_dealt_damage, allied_drones_lost, etc.

- **Unlock Data Structures** (lines 1430-1671+): Individual unlock entries with:
  - `tier`: 1, 2, or 3 (progression level)
  - `icon`: Resource path (e.g., `res://assets/images/random_pics/main_icon2.png`)
  - `description`: Human-readable unlock text describing the upgrade
  - `passive_unlocks`: Array of passive upgrades triggered by the unlock

  **Upgrade Types Documented**:
  - Combat: `Laser Delay`, `Shield Health`, `Shield Regen`, `Mining Power`
  - Ship Limits: `Drone Limit`, `Drone Respawn`, `Support Limit`, `Frigate Limit`
  - Production: 1% production speed improvements with diminishing returns

### 2. How Unlock Definitions Are Grouped

Unlocks are organized by **ship type** into three categories (lines 8844-8852):

```gdscript
match category:
    "drone_unlocks":
        spawner.custom_class = "drone"
        source = Drones
    "support_unlocks":
        spawner.custom_class = "support"
        source = Supports
    "frigate_unlocks":
        spawner.custom_class = "frigate"
        source = Frigates
```

Each group contains:
- Tier-based progression (tiers 1-3)
- Associated passive_unlocks arrays
- Ship-type-specific upgrade effects

### 3. Systems That Depend on Unlocks.gd

Three primary systems reference unlock data via the spawner system:

| System | Source Constant | Custom Class | Role |
|--------|-----------------|--------------|------|
| Drones | `Drones` | `"drone"` | Handles drone unlock progression, damage modifiers, drone limit upgrades |
| Supports | `Supports` | `"support"` | Handles support unlock progression, shield health/regen, support limit upgrades |
| Frigates | `Frigates` | `"frigate"` | Handles frigate unlock progression, laser damage, frigate limit upgrades |

The spawner system uses `spawner.custom_class` to route unlock effects to the appropriate ship type, with modifier processing in the `elif (category == "drone_unlocks")` block (lines 8900+).
